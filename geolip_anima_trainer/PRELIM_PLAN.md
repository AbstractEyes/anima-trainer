# Anima Trainer — Repo Setup Brief

Handoff for Claude Code to scaffold a repo that finetunes **CircleStone Labs Anima**
(2B anime text-to-image) using **tdrussell/diffusion-pipe**. This file can become the
repo's `CLAUDE.md`.

## Objective
Finetune Anima on a slice of an HF `datasets`-format parquet repo. diffusion-pipe is the
only trainer that supports Anima (the DiT is NVIDIA Cosmos-Predict2-2B), so we use it and
*bridge* the dataset into the directory layout it requires — we do NOT switch trainers.

## Environment (fixed facts — do not re-derive)
- GPU: **RTX PRO 6000 Blackwell, 96 GB, sm_120**. A 2B model is tiny here.
- **bf16 throughout. No fp8** (e4m3 degrades this lineage; 96 GB makes fp8 pointless).
- **Do NOT install flash-attn** — non-functional on this sm_120 box. diffusion-pipe runs on
  **SDPA**; that is the correct/expected backend.
- No block swapping, no activation checkpointing needed (VRAM is abundant).
- Python 3.12, conda env recommended.

## Dataset
- Repo: `AbstractPhil/diffusion-pretrain-set-ft1`
- Config to train on: **`qwen_90k`** — 83.0K rows, 17 cols, 101.1 GB, ~1.22 MB/row (high-res,
  good for 1024). Uniform 17-col schema shared with all configs.
- Captions are **JSON**, not strings: `caption_animetimm_json` (booru-style),
  `caption_vlm_json`, `captions_source_json`. The bridge renders them to tag strings.
- Quality gates in-schema: `audit == "approved"`, `age_classifier_pass == True` (on by default
  in the bridge). **These may be unpopulated for the qwen_90k source — verify first.**

## Setup steps (in order)
1. `git clone --recurse-submodules https://github.com/tdrussell/diffusion-pipe external/diffusion-pipe`
2. `conda create -n anima python=3.12 -y && conda activate anima`
3. `pip install -r external/diffusion-pipe/requirements.txt && pip install datasets`
   - Do not add flash-attn.
4. `python scripts/download_anima.py --dest models/anima --base base-v1.0`
   - paste the printed paths into `configs/anima_lora.toml` `[model]`.
5. **Inspect the source BEFORE extracting** (the HF viewer can't scan this config):
   ```
   python scripts/hf_to_diffusion_pipe.py --repo AbstractPhil/diffusion-pretrain-set-ft1 \
       --configs qwen_90k --inspect
   ```
   Read off: which caption column is filled, and whether the audit/age gates are populated.
6. Extract (transient) — fill `<caption-col>`/`<fmt>` from step 5, add `--no-age-filter` /
   `--no-audit-filter` if those gates are empty:
   ```
   python scripts/hf_to_diffusion_pipe.py --repo AbstractPhil/diffusion-pretrain-set-ft1 \
       --configs qwen_90k --out datasets/anima_qwen90k \
       --caption-column <caption-col> --caption-format <fmt> \
       --route-by source --limit-per-concept 8000
   ```
7. Build the dataset toml:
   ```
   python scripts/build_multiconcept_dataset.py --root datasets/anima_qwen90k \
       --out configs/anima_dataset.toml
   ```
   - If `source` is uniform (single bucket), this is a single-concept run — see decision below.
8. Cache, then train:
   ```
   cd external/diffusion-pipe
   PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True \
     deepspeed --num_gpus=1 train.py --deepspeed --config ../../configs/anima_lora.toml --cache_only
   # verify cache OK, then drop --cache_only to train
   ```
9. After caching succeeds, the `datasets/anima_qwen90k` scratch dir can be deleted — training
   reads from the cache. Steady-state footprint = cache + the archival parquet.

## Open decisions (resolve from step 5 output)
- **Caption column**: default `caption_animetimm_json`/`animetimm`. Switch to `vlm` or `source`
  if animetimm is empty for this source.
- **Single vs multi-concept**: qwen_90k is one coherent source. If its `source` column is
  uniform → **single-concept LoRA**: skip the balancer, point one `[[directory]]` at the
  extracted folder, drop `[adapter]` rank to 32, keep `llm_adapter_lr = 0`. Only keep rank 64
  + multi-directory balancing if `source` genuinely subdivides.

## Non-negotiable rules
- **`llm_adapter_lr = 0`** (adapter frozen). High knowledge density, degrades easily. Only
  consider raising it (≤5e-6) for many genuinely-new concepts, A/B'd against a frozen baseline.
- **Anima is tag-order sensitive**: quality → subject → character → series → @artist → general.
  `shuffle_caption = false`.
- **License: non-commercial.** Anima is CircleStone NC + inherits the NVIDIA Open Model License
  (Cosmos derivative). Any LoRA/finetune is a derivative — non-commercial only.
- diffusion-pipe input = directories of image files + matching `.txt` sidecars. It uses the
  `datasets` library only as an internal latent/embedding cache, never as an input format.
